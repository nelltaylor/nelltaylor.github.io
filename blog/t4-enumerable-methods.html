Phase 0 Week 4 Technical Blog: Enumerable Methods By: Nell Taylor | 11-2-14

Map is an enumerable method that does two very important things: 1) It alters every element of an array when combined with block of code and another method (optional) that instructs it on how those elements should be altered. 2) It returns a new array of those altered elements, rather than changing them in the original array. Therefore, map is a non-destructive method that can be incredibly powerful for increasing the functionality of other methods and producing new arrays for you to work with. (And, if you just want to have #1 without the new array, you can use map!, the destructive version.) Say that you had an array of family members' names, but you were all going into witness protection and wanted to see test out some different last names to see which ones you like best. You don't want to alter the original array because then you wouldn't be able to keep modifying it as many times or as easily as you'd like. You could do something like this: names = ["John", "Jane", "Billy", "Suzy"].map{ |a| a + " Mermelstein" }
#=> John Mermelstein Jane Mermelstein Billy Mermelstein Suzy Mermelstein
